## Что такое Работяга

Работяга — Telegram Mini App для управления сменами в баре. Открывается прямо внутри Telegram через кнопку бота, но также работает в браузере (без пушей и Telegram-привязки).

Приложение решает три задачи:
- **Операционный контроль** — все задачи смены в одном месте, видно кто что сделал и когда.
- **Дисциплина** — система карточек фиксирует нарушения с историей.
- **Аналитика** — выручка, процент выполнения задач, тренды по каждому сотруднику.

---

## Вход и авторизация

При первом открытии приложение показывает экран выбора имени. Каждый аккаунт защищён паролем.

**Первый вход:** введи пароль дважды — он будет установлен автоматически.  
**Последующие входы:** вводишь тот же пароль.  
**Забыл пароль:** попроси управляющего или разработчика сбросить пароль через вкладку «Команда».

На экране входа отображаются:
- Статус подключения к серверу (зелёный = данные синхронизируются между устройствами).
- Статус Telegram (подключён ли ID — нужно для пушей).
- Текущий статус каждого сотрудника из расписания (цветная точка).

> Сессия хранится в httpOnly cookie и восстанавливается автоматически при следующем открытии. Явный выход — кнопка с именем в правом верхнем углу.

## Роли и права

| Роль | Задачи | Расписание | Команда | Выдача карточек | Управление |
|---|---|---|---|---|---|
| Бармен | Свои | Просмотр | — | — | — |
| Шеф-бармен | Все + создание | Просмотр | Статистика | — | — |
| Управляющий | Все + создание | Редактирование | Полное | ✓ | ✓ |
| Разработчик | Всё | Всё | Всё | ✓ | + системные |

Права каждого аккаунта настраивает управляющий или разработчик во вкладке «Команда» → кабинет сотрудника → Права доступа.

## Интерфейс

### Навигация

В верхней части — шапка с названием, датой и вкладками. Вкладки можно **перетаскивать** (drag-and-drop) — порядок сохраняется на устройстве.

Справа в шапке:
- **📥 Инбокс** — личные задания, назначенные тебе (бейдж с непрочитанными).
- **Статус смены** — чип с текущим статусом (на смене / отработал / выходной и т.д.).
- **Тема** — переключатель авто / светлая / тёмная.
- **Имя + точка** — выход и статус сервера (зелёная = онлайн, красная = офлайн).

Управляющий и разработчик также видят кнопки **«Управление»** и **«Админка»** в шапке.

### Тосты и уведомления

Мгновенные сообщения всплывают вверху страницы. Нажми на тост, чтобы закрыть его досрочно.

---

## Вкладка «Сегодня»

Главный экран смены. Открывается при входе.

### Прогресс-бар

Показывает сколько задач дня выполнено: `X из Y` и процент. Обновляется в реальном времени.

### Состав смены

Карточки сотрудников с цветовым статусом. Управляющий может нажать на имя и открыть личный кабинет сотрудника.

⚠ Если штат не соответствует норме (например, в пятницу после 18:00 нужно 3 человека) — появляется предупреждение.

### Выручка

Блок «Выручка дня» — управляющий вводит план и факт. Данные сохраняются в расписание дня и видны в истории.

### Гоу-лист

Общий список дел смены — что нужно сделать за день помимо основных задач (заказать что-то, позвонить, разобраться с ситуацией). Редактирует вся команда:
- Добавить пункт — поле внизу блока.
- Отметить выполненным — нажать на пункт.
- Удалить — кнопка корзины.

Гоу-лист не связан с основной задачной системой и не влияет на закрытие смены.

### Задачи дня

Список задач, актуальных для сегодня по роли/назначению. Активные задачи вверху, выполненные — в аккордеоне «Выполнено».

**Управление порядком задач:** шеф-бармен и управляющий могут перетаскивать активные задачи.

**Действия с задачей:**
- Свайп влево / кнопка ✓ — выполнить / отменить.
- Длительное нажатие или кнопка ✏ — редактировать (управляющий/шеф).
- Кнопка передачи — перенести на завтра с заметкой.

### Нерегулярные задачи (бэклог)

Задачи без расписания («требует внимания») показываются отдельным разделом внизу. Выполняются один раз и не повторяются.

### Передачи смены

Заметки, оставленные предыдущей сменой, отображаются в блоке «Передача от прошлой смены».

---

## Вкладка «Задачи»

Полный список всех задач системы (для шеф-бармена и управляющего).

**Фильтры:** по повторению (ежедневные / открытие / закрытие / и др.), по статусу (активные / архивные).

**Создание задачи** — кнопка «+» (FAB) или через карточку задачи → редактировать.

**Поля задачи:**
- Название
- Тип повторения: открытие, закрытие, ежедневно, по будням, еженедельно, однократно, нерегулярная
- Дата (для однократных и нерегулярных)
- Время напоминания
- Исполнитель: смена (все) или конкретный сотрудник
- Назначить конкретному человеку (с отслеживанием в инбоксе)
- Заметки
- Признак «отчёт» (только для сотрудников с флагом `report` в смене)

**Архивирование** — скрывает задачу из системы без удаления истории.

---

## Вкладка «График»

Календарь смен на месяц.

### Просмотр

- Каждый день показывает кто выходит, статус штата, метку праздника/события.
- Цвет дня — от зелёного (норма) до красного (нехватка персонала).

### День в деталях

Нажми на день — откроется детальный экран:
- Список смен с временем начала/окончания.
- Задачи этого дня.
- Выручка (план/факт) — управляющий может ввести/изменить.
- Передачи смены.

**Управление сменами** (управляющий / шеф):
- Добавить сотрудника в смену.
- Убрать из смены.
- Изменить время смены.

### Статусы сотрудников

| Статус | Значение |
|---|---|
| На смене | Сейчас работает |
| Смена сегодня | Смена началась позже / ещё не закончилась |
| Отработал | Смена завершена |
| Выходной | Не в расписании на сегодня |
| Отпуск | Ручное переопределение |
| Больничный | Ручное переопределение |

## Вкладка «Команда»

Доступна управляющему, шеф-бармену и разработчику.

### Состав команды (управляющий / шеф)

- Добавить нового сотрудника.
- Удалить из списка.
- Открыть личный кабинет сотрудника.

### Статистика команды

Карточки по каждому сотруднику:
- Процент выполнения задач за 14 и 30 дней.
- Тренд (растёт / падает / стабильный).
- Активные карточки дисциплины.
- Флаги подозрительной активности (если паттерн кажется нестандартным).

### Управление аккаунтами

- Задать/сбросить пароль сотруднику.
- Изменить роль.
- Настроить права доступа (ACL).

### Журнал событий

Хронологический лог всех действий: входы, выходы, выполнение задач, выдача карточек, изменения расписания. Хранится последние 500 событий.

## Личный кабинет

Открывается через кнопку **«Управление»** в шапке (для всех ролей — свой кабинет) или через «Команда» → имя сотрудника (для управляющего — чужой кабинет).

**Разделы:**

### Обзор
- Роль, текущий статус смены.
- Активные карточки дисциплины.
- Статистика: процент выполнения за 14 и 30 дней, тренд.
- Часы за текущий месяц и неделю.
- Предстоящие смены.
- Автоматические рекомендации системы.

### Задачи
- Задачи, назначенные этому сотруднику.
- Отметить выполненными прямо из кабинета.

### Настройки
- Изменить пароль (текущий + новый).
- Управляющий может выдать карточку из кабинета сотрудника.

---

## Личный инбокс

Иконка 📥 в шапке (только для не-управляющих). Показывает задачи, которые были **назначены тебе лично** (поле «Назначить»).

- Бейдж с числом непрочитанных исчезает при открытии инбокса.
- Задачи можно отметить выполненными прямо из инбокса.
- Видны все назначенные задачи — не только на сегодня.

---

## Передача смены

Доступна из кнопки на карточке задачи (значок «передать»).

Позволяет оставить заметку для следующей смены и опционально создать задачу-перенос на завтра с пометкой `[Перенос]`.

При закрытии смены (все задачи выполнены) — модальное окно предлагает перенести **все невыполненные** задачи разом.

---

## Закрытие смены

Когда все регулярные задачи дня выполнены, день считается «закрытым»:
- Прогресс-бар показывает 100%.
- Появляется зелёный баннер «Смена закрыта».

После **23:30** автоматически открывается модальное окно итогов смены:
- Сводка: выполнено X из Y задач.
- Невыполненные задачи (если есть) с кнопкой «Перенести на завтра».
- Кнопка «Закрыть» — фиксирует факт закрытия.

> Telegram-пуш с итогами отправляется управляющему и шеф-бармену автоматически при закрытии после 23:30.

## Карточки (дисциплина)

Система прогрессивных взысканий. Выдаётся управляющим или разработчиком.

| Тип | Значение |
|---|---|
| 🟡 Жёлтая | Предупреждение |
| 🟠 Оранжевая | Серьёзное нарушение |
| 🔴 Красная | Критическое нарушение |

**Прогрессия:** система автоматически повышает тип карточки при накоплении (2 жёлтых → оранжевая, 2 оранжевых → красная).

**Конфиденциальность:** при выдаче можно отметить «конфиденциально» — карточка не показывается в публичной статистике команды.

**История** хранится в личном кабинете сотрудника. Неактивные (снятые) карточки помечаются отдельно.

## Telegram-пуши

Пуши работают только при открытии приложения через бота Telegram (нужна привязка Telegram ID к имени в системе — происходит автоматически при первом входе через Telegram).

| Время | Событие |
|---|---|
| 20:00 | Напоминание за день до смены |
| 09:00 | Личные задачи на сегодня |
| 22:00 | Напоминание закрыть смену (если не закрыта) |
| По событию | Итоги смены при закрытии после 23:30 |

**Шаблоны пушей** настраиваются через Админку (доступна управляющему и разработчику).

> Пуш не отправляется, если пользователь заблокировал бота — система это обнаруживает и не повторяет попытку. Статистику отправок видно в Админке → Статистика пушей.

## Тема оформления

Кнопка в правом верхнем углу шапки. Три режима:
- 🖥 **Авто** — следует системной теме устройства, меняется без перезагрузки.
- ☀️ **Светлая** — всегда светлая.
- 🌙 **Тёмная** — всегда тёмная.

Выбор сохраняется на устройстве.

---

## Синхронизация данных

Все данные хранятся на сервере (KV-хранилище в `data.json`). Локальное хранилище (`localStorage`) — кеш/фолбек при потере связи.

- **Зелёная точка** рядом с именем = сервер доступен, данные синхронизируются между всеми устройствами.
- **Красная точка** = сервер недоступен. Данные сохраняются только локально до восстановления связи.

Сервер опрашивается каждые 15 секунд.

## Планируемые функции

> Раздел обновляется по мере реализации.

### 📊 Аналитика смен (в разработке)
Блок под календарём в «График»:
- Итоги месяца по выручке (план vs факт) с графиком по дням.
- Статистика выполнения задач по неделям.
- Интеграция с iiko — импорт реальной выручки автоматически.

### 🏖 Управление отпусками (планируется)
Личный кабинет → раздел «Отпуска»:
- Сотрудник подаёт заявку на отпуск с датами.
- Управляющий согласует или отклоняет.
- При согласовании статус в расписании меняется автоматически.
- Пуш-уведомление сотруднику о решении.

### 📈 Личная статистика расширенная (планируется)
- График выполнения задач за 3 месяца.
- Сравнение с командой.
- Рейтинговая таблица смены.

### 🍺 Интеграция с iiko (планируется)
- Автоматический импорт выручки за день.
- Сравнение выручки с нормой по дню недели.
- Экспорт отчёта смены.

### 🔔 Расширенные пуши (планируется)
- Настраиваемое время отправки.
- Дополнительный тип пуша: итоги недели по команде для управляющего.
- Отказ от пушей на конкретный день (например, выходной).

### 👤 Расширенный профиль сотрудника (планируется)
- Фото аватара.
- Контактный телефон.
- Дата начала работы.
- Документы (для внутреннего пользования).

### 🌐 SaaS-режим (долгосрочно)
- Мультиплейс: несколько заведений в одном аккаунте.
- Онбординг без участия разработчика.
- Биллинг и тарифные планы.
