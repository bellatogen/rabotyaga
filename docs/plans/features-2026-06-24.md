## Item 1: Rich Events (события) — events:v2
**Goal**: Upgrade events from flat `{date: string}` KV to full rich event objects.  
**Done when**:
- `events:v2` KV stores array of event objects with all fields
- `EventsTab.jsx` fully rewritten: create/edit form with all fields
- Recurrence computed correctly (daily/weekly/every-N/weekday/once)
- Migration: `events:v1` → `events:v2` on first load
- Recurring events visible in Today tab (if event applies today)
- `server.js` updated: `events:v2` in MANAGER_ONLY_KV, `events:v1` kept for compat

**Event schema**:
```js
{
  id: string,           // nanoid
  title: string,        // название
  type: string,         // id из EVENT_TYPES
  description: string,  // контекст/описание
  // Дата или период
  startDate: string,    // YYYY-MM-DD
  endDate: string|null, // YYYY-MM-DD или null (однодневное)
  // Повторяемость
  recurrence: {
    type: 'once'|'daily'|'weekly'|'every_n'|'weekday',
    interval: number|null,   // для every_n — каждые N дней
    weekday: number|null,    // 0-6 для weekday (0=вс)
    endDate: string|null,    // до какой даты повторять
  },
  // Ответственные
  responsible: string[],    // имена из staff
  // Тайминг
  timing: { start: string, end: string }, // "18:00", "22:00"
  // Локация
  location: {
    type: 'own'|'external',
    address: string|null,
    transferDetails: string|null,
  },
  // Смета (опциональная)
  budget: {
    enabled: boolean,
    items: [{name:string, amount:number}],
  },
  // Маркетинг
  marketing: {
    posts: [{
      platform: string,   // 'vk'|'instagram'|'telegram'|'другое'
      scheduledDate: string,
      responsible: string,
      pushEnabled: boolean,
    }]
  },
  // Аналитика/отчёт
  analytics: {
    notes: string,
    report: string,
  },
  createdAt: string,
}
```

**Key files**: `frontend/src/pages/EventsTab.jsx`, `frontend/src/constants/events.js`, `frontend/src/App.jsx` (eventsData state, ld/sv calls), `rabotyaga-bot/server.js` (MANAGER_ONLY_KV)

**Status**: [x] complete — build clean, no new lint errors

## Item 3: Bot macros — message to external chats
**Goal**: Write to external Telegram chats from bot on schedule (macros).  
**Done when**:

- KV key `bot_macros:v1` stores macros `[{id, name, chatId, template, schedule:{type,interval,weekday,time}, active}]`
